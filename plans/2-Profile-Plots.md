# Profile Plotting

## Purpose
Create a profile plotting feature for profile-type parts in the AutoCAD plugin.

A profile plot is a process sheet used to manufacture a profile part. Unlike
plate parts, which are handled through nesting, profile parts should generate a
dedicated drawing/plot sheet that shows a scaled representation of the part
inside a fixed plotting area.

Example:
A structural T profile is cut to 300 mm long and includes one or more end cuts.
The generated profile plot should show the profile in a scaled view, fit it
within a 1000 x 400 mm plot area, and display all required manufacturing
dimensions and angles needed to create the part.

The plugin already has support for profile-type drawings. This feature should
build on that existing functionality and generate the plot/process sheet for the
profile.

## Current Scope
- Generate a profile process sheet for a profile-type part
- Use the existing profile drawing functionality as the source geometry
- Create a scaled drawing of the part within a 1000 x 400 mm plotting area
- Add dimensions for overall length and relevant cut geometry
- Add angles where required for angled cuts
- Produce a plot-ready drawing/layout for manufacturing

## Future Scope
- Batch-generate profile plots for multiple selected parts
- Export profile plots to PDF automatically
- Add title block / revision / approval metadata
- Add labels for material, profile type, stock size, and part mark
- Support more advanced annotations for holes, copes, bevels, weld prep, etc.

## Technical Context
- Project type: C# AutoCAD extension
- Existing system already supports profile-type drawings
- This feature should extend the current profile drawing workflow rather than
  replace it
- Follow existing plugin architecture and command patterns
- Reuse existing geometry extraction and drawing logic where possible
- Do not break current profile drawing behavior

## Domain Context
A part in the system can be either:
- Plate part
- Profile part

Plate parts are handled through nesting workflows.

Profile parts require a profile plot/process sheet showing how the profile
should be cut and prepared for production.

A profile part may include:
- A profile section/type (example: structural T, angle, flat bar, channel, etc.)
- A total cut length
- End cuts on one or both ends
- Optional angled cuts
- Other manufacturing geometry already represented in the existing profile
  drawing logic

## Goal
For a selected profile part, generate a clear process sheet that:
- shows the part in a scaled view
- fits the drawing into a 1000 x 400 mm plotting region
- displays dimensions required by the shop to fabricate the part
- displays angle annotations where necessary
- is readable and plot-ready

## Assumptions
- The geometry for the profile part already exists or can already be generated by
  the current plugin
- The profile plot will be generated from the existing profile drawing data
- Units are millimeters
- The process sheet is intended for fabrication use
- The scaled representation does not need to be full-size; it only needs to be
  accurate and clearly dimensioned

## Plot Area Requirements
The profile plot must fit within a fixed drawing region of:
- Width: 1000 mm
- Height: 400 mm

Requirements:
- Automatically determine a scale factor that fits the profile drawing inside
  the 1000 x 400 mm region
- Maintain aspect ratio
- Center the plotted profile within the usable region, or align using a
  consistent layout rule
- Leave room for dimensions and annotations so text does not run outside the
  plot area
- If necessary, use margins inside the 1000 x 400 mm region

Suggested implementation detail:
- Define an inner usable area with margins, for example:
  - left/right margin
  - top/bottom margin
- Fit actual geometry to the usable area, not directly to the outer boundary

## Geometry Requirements
The feature should use the existing profile drawing representation and generate a
simplified plot view suitable for manufacturing.

The plotted geometry should include:
- Overall profile outline or centerline representation, depending on existing
  profile drawing conventions
- End cut geometry
- Any angled cut geometry that affects fabrication
- Any additional profile features already supported by the profile drawing logic
  that are relevant to manufacturing

Open questions to resolve in implementation:
- Should the process sheet show a side/profile view only, or multiple views?
- Should the cross-sectional profile shape also be shown, or just the cut
  geometry along length?
- Should hidden/detail geometry be omitted to keep the sheet readable?

Default recommendation:
- Use the simplest single-view representation that clearly communicates how to
  cut the profile
- Only include additional views if required for manufacturing clarity

## Dimensioning Requirements
The generated profile plot should include all dimensions needed to fabricate the
part.

Minimum required dimensions:
- Overall part length
- End cut lengths/offsets where applicable
- Angle callouts for angled cuts
- Any additional dimensions required to fully define the end cut geometry

Dimensioning rules:
- Dimensions should be placed outside the profile geometry when possible
- Dimensions should not overlap each other or the part geometry
- Text height and arrow sizes should scale consistently for readability
- Dimension placement should remain readable after automatic scaling
- Do not place unnecessary duplicate dimensions
- If the part is a simple straight cut with no angled ends, only show the
  essential dimensions

Angle rules:
- Show angle annotations when a cut is not square/perpendicular
- Clearly indicate which end the angle applies to
- Use a consistent convention for angle measurement

Open questions to define:
- Should dimensions be true AutoCAD dimensions, text/line annotations, or a
  combination?
- Should angle values be rounded, and to how many decimal places?
- What dimension style should be used?

Suggested defaults:
- Use standard AutoCAD dimension entities where practical
- Use the project’s existing dimension style if one exists
- Round angle values to a consistent precision, such as 0.1 degrees or 0.5
  degrees depending on shop preference

## Layout Requirements
The process sheet should be organized in a predictable way.

Suggested layout:
- Main plotted profile centered in the primary drawing area
- Dimensions arranged above, below, or to the sides based on available space
- Optional metadata block in a reserved area
- Optional title/process information area if the plugin already has a title
  block pattern

Potential metadata to show on the sheet:
- Part name / part mark
- Profile type
- Material
- Length
- Scale used for plotting
- Drawing date
- Revision if available

If metadata is not part of current scope, leave the layout extensible so it can
be added later.

## Data Requirements
The profile plotting feature may require access to:
- Part ID
- Part name / mark
- Part type
- Profile type / section
- Profile length
- End cut geometry
- Angle information
- Existing profile drawing geometry
- Material metadata if available

If some of this data is not currently stored in a normalized way, identify what
the existing plugin already uses and reuse that source.

## Workflow
Suggested user workflow:
1. User selects or identifies a profile part
2. Plugin loads the existing profile drawing data for that part
3. Plugin determines the bounding box of the drawable geometry
4. Plugin calculates a scale factor to fit within the 1000 x 400 mm plot region
5. Plugin places the scaled geometry into the process-sheet layout
6. Plugin adds required dimensions and angle annotations
7. Plugin creates or updates the output drawing/layout
8. User can review and plot/export the result

## Scaling Rules
- Compute the extents of the profile drawing geometry
- Apply a uniform scale factor so the full drawing fits inside the usable plot
  area
- Preserve proportions
- Account for dimension text and annotation spacing when calculating final
  placement
- If the profile is very small, avoid scaling so large that dimensions become
  awkward
- If the profile is very large, prioritize full visibility and dimension clarity

Suggested implementation detail:
- Separate geometry scale from annotation scale if needed
- Keep annotation sizes readable and consistent regardless of part length

## Annotation Rules
The plot should remain readable for manufacturing.

Rules:
- Use consistent text height
- Use consistent line weights/layers based on project standards
- Place dimensions in a way that avoids crossing extension lines where possible
- Add angle labels only where needed
- Add cut labels if the cut type cannot be understood from dimensions alone

Potential future annotations:
- Left end / Right end markers
- Datum/reference edge
- Cut sequence notes
- Shop notes

## Output Requirements
The feature should generate a plot-ready output.

Possible output targets:
- AutoCAD layout
- AutoCAD drawing space region reserved for profile plots
- PDF export in future scope

Minimum output requirement:
- Create a drawing or layout that can be plotted directly by the user

Open questions:
- Should each profile plot be created in model space or paper space?
- Should a new layout be created per part?
- Should the output overwrite an existing plot or create a new one?

Recommended default:
- Create a dedicated layout or drawing area per profile part using the plugin’s
  existing drawing conventions

## Error Handling
Handle these situations gracefully:
- Part is not a profile part
- No existing profile geometry is available
- End cut geometry is incomplete
- Part geometry cannot be dimensioned automatically
- Part extents exceed expected limits
- Annotation placement fails due to insufficient space

In error cases:
- Show a clear user-facing message explaining what data is missing
- Avoid creating partial or misleading process sheets when critical data is
  unavailable
- Allow fallback generation of geometry-only output if dimensioning cannot be
  completed, if that is acceptable

## Acceptance Criteria
- User can generate a profile plot for a profile-type part
- The plot uses the existing profile drawing as the source
- The resulting geometry fits within a 1000 x 400 mm plotting area
- The drawing is scaled proportionally
- Overall part length is shown
- End cuts are shown and dimensioned where applicable
- Angles are shown when required
- The generated sheet is readable and suitable as a fabrication process sheet
- Existing profile drawing functionality remains intact

## Implementation Notes
Suggested implementation phases:

### Phase 1: Basic Plot Generation
- Identify profile part input
- Reuse existing profile drawing generation
- Calculate geometry extents
- Scale and place geometry into the plot area

### Phase 2: Dimensioning
- Add overall length dimensions
- Add end cut dimensions
- Add angle annotations
- Tune text placement for readability

### Phase 3: Layout/Polish
- Standardize margins and layout positioning
- Add metadata fields if desired
- Improve layer/style consistency
- Prepare for future PDF export or batch generation

## Questions To Resolve Before Implementation
- What exact profile types must be supported first?
- What existing object or data structure represents a profile part?
- What existing function/class already creates the profile drawing?
- Should the plot show one view or multiple views?
- Should dimensions use AutoCAD dimension entities or custom annotations?
- What dimension style/layer/text height standard should be used?
- Where should the generated process sheet live: model space, paper space, or a
  dedicated drawing?
- Should the command create a new plot every time or update an existing one?
